## 二、核心发现：卡片之间的数据流关系

### 发现 1：存在天然的「输出→输入」管道

很多卡片的输出，恰好是另一个卡片的输入。这意味着用户可以把卡片 A 的输出直接粘贴到卡片 B 的输入框，形成**无缝的科研工作流管道**。

### 发现 2：有 3 类卡片可以接受「论文本身」作为输入

- **论文精读笔记**：直接粘贴论文内容
- **创新点提炼**：可以粘贴论文草稿的核心描述
- **逐段语言审查**：直接粘贴论文的英文 LaTeX 段落

这意味着用户读到一篇好论文时，可以直接把论文内容喂给这些卡片，不需要额外加工。

## 三、完整的数据流图（按阶段）

### 阶段 1 → 阶段 2：调研产出驱动 Idea 生成

```
文献检索策略 ──→ 检索结果（论文列表）
                     │
                     ▼
论文精读笔记（×N篇）──→ 结构化笔记
                     │
                     ├──→ 领域图谱梳理（代表性论文信息）
                     │         │
                     │         ▼
                     │    领域图谱（技术演进 + 流派 + 空白）
                     │
                     └──→ 研究 Idea 头脑风暴（已有工作信息）
```

**具体粘贴路径：**
- 「文献检索策略」的输出「关键论文清单」→ 喂给「论文精读笔记」逐篇精读
- 多篇「论文精读笔记」的输出 → 汇总喂给「领域图谱梳理」
- 多篇「论文精读笔记」的输出 + 「领域图谱梳理」的输出「研究空白」→ 喂给「研究 Idea 头脑风暴」

### 阶段 2 内部：Idea 的筛选漏斗

```
研究 Idea 头脑风暴 ──→ Top-2 Idea 推荐
        │                    │
        ▼                    ▼
   可行性评估            创新点提炼
   （逐个评估）          （逐个提炼）
        │                    │
        └────── 合并 ────────┘
                 │
                 ▼
          确定最终 Idea → 进入阶段 3
```

**具体粘贴路径：**
- 「研究 Idea 头脑风暴」输出的每个 Idea → 分别喂给「可行性评估」和「创新点提炼」
- 「可行性评估」的结论 + 「创新点提炼」的强化策略 → 综合决策

### 阶段 2 → 阶段 3：从 Idea 到方法设计

```
创新点提炼（核心创新点）
        +
研究 Idea 头脑风暴（核心方法描述）
        +
领域图谱梳理（差异化定位）
        │
        ▼
  方法架构设计 ──→ 完整技术方案
        │
        ├──→ 损失函数与优化策略（方法架构作为输入）
        │
        └──→ 概念图设计指引（方法概述 + 数据流）
```

**具体粘贴路径：**
- 「创新点提炼」的创新点 + 「Idea 头脑风暴」的方法描述 → 喂给「方法架构设计」
- 「方法架构设计」的输出 → 直接喂给「损失函数与优化策略」和「概念图设计指引」

### 阶段 3 → 阶段 4：从方法到实验

```
方法架构设计 ──→ 完整技术方案
        │
        ├──→ 实验方案设计（方法 + 核心贡献点）
        │
        └──→ 消融实验设计（方法架构 + 核心模块）
```

**具体粘贴路径：**
- 「方法架构设计」的输出 → 喂给「实验方案设计」和「消融实验设计」
- 「创新点提炼」的贡献点 → 也应喂给「实验方案设计」（验证贡献）

### 阶段 4 → 阶段 5：从实验到论文

```
实验结果分析 ──→ LaTeX 分析段落
        │
        ▼
  ┌─────────────────────────────────────────┐
  │           论文写作（5张卡片）              │
  │                                          │
  │  Abstract ← 研究背景 + 方法概述 + 实验结论 │
  │  Introduction ← 背景 + 问题 + 方法 + 贡献点 │
  │  Related Work ← 领域图谱 + 精读笔记       │
  │  Method ← 方法架构设计的全部输出           │
  │  Experiments ← 实验方案 + 结果分析        │
  └─────────────────────────────────────────┘
```

**具体粘贴路径：**
- 「方法架构设计」→ 喂给「Method 章节撰写」
- 「实验方案设计」+ 「实验结果分析」→ 喂给「Experiments 章节撰写」
- 「领域图谱梳理」+ 多篇「论文精读笔记」→ 喂给「Related Work 撰写」
- 「创新点提炼」的贡献点 → 喂给「Introduction 撰写」
- 所有论文写作卡片的输出 → 汇总为完整论文

### 阶段 5 → 阶段 6：审稿与修改

```
论文全文（5个章节的 LaTeX）
        │
        ├──→ 论文一致性审查（Intro贡献 + Method + Experiments）
        │
        └──→ 逐段语言审查（逐章审查英文）
```

**具体粘贴路径：**
- 「Introduction」的贡献声明 + 「Method」摘要 + 「Experiments」结果 → 喂给「论文一致性审查」
- 每个章节的 LaTeX → 逐段喂给「逐段语言审查」

### 阶段 6 → 阶段 7：投稿准备

```
论文全文（修改后）
        │
        ├──→ 投稿 Checklist 核对
        ├──→ 论文压缩与页数控制
        └──→ Rebuttal 撰写（收到审稿意见后）
```

## 四、哪些卡片可以直接接受「论文原文」

| 卡片 | 可接受的论文内容 | 说明 |
|------|-----------------|------|
| **论文精读笔记** | 完整论文（摘要+方法+实验+结论） | 最自然的论文输入卡片，设计初衷就是精读 |
| **创新点提炼** | 论文草稿的核心描述 | 可以在写论文阶段把草稿喂进去提炼 |
| **逐段语言审查** | 英文 LaTeX 论文段落 | 可以把整篇论文分段喂入 |
| **论文一致性审查** | 各章节内容（Intro+Method+Experiments） | 需要拆分后分别粘贴 |

## 五、推荐的典型工作流（端到端）

### 工作流 A：从零开始做一篇论文

```
1. 输入研究关键词 → 「文献检索策略」
2. 按检索策略找论文 → 逐篇喂给「论文精读笔记」
3. 精读笔记汇总 → 喂给「领域图谱梳理」
4. 图谱 + 笔记 → 喂给「研究 Idea 头脑风暴」
5. 候选 Idea → 分别喂给「可行性评估」+「创新点提炼」
6. 确定 Idea → 喂给「方法架构设计」
7. 架构设计 → 喂给「损失函数」+「概念图」+「实验方案」+「消融实验」
8. 跑完实验 → 结果喂给「实验结果分析」
9. 方法+实验+创新点 → 分别喂给 5 张论文写作卡片
10. 论文初稿 → 喂给「一致性审查」+「语言审查」
11. 修改后 → 喂给「投稿 Checklist」+「论文压缩」
```

### 工作流 B：精读一篇论文并提炼 Idea

```
1. 论文全文 → 「论文精读笔记」→ 获得结构化理解
2. 精读笔记 → 「研究 Idea 头脑风暴」→ 从这篇论文出发找新方向
3. 候选 Idea → 「可行性评估」→ 筛选
4. 可行的 Idea → 「创新点提炼」→ 强化 novelty
```

### 工作流 C：论文写完后的审稿准备

```
1. 论文各章节 → 「一致性审查」→ 发现前后不一致
2. 论文各章节 → 「语言审查」→ 逐段润色
3. 修改后的论文 → 「投稿 Checklist」→ 格式合规检查
4. 如果超页 → 「论文压缩」→ 精简方案
```

## 六、潜在的改进方向（仅分析，不改文件）

1. **「Related Work」的输入可以更明确**：目前输入是「研究方向 + 相关工作列表 + 方法创新点」，但实际上「领域图谱梳理」的输出（技术演进 + 流派 + 代表性工作）几乎可以直接作为 Related Work 的输入骨架。

2. **「Introduction」的输入与「创新点提炼」高度重合**：两者都需要贡献点，「创新点提炼」的输出可以直接喂给「Introduction」。

3. **阶段 5 的写作卡片之间存在依赖**：Method 章节需要先写完才能写 Experiments（因为要引用方法细节），Abstract 通常最后写（需要全文概览）。这不是卡片设计的问题，而是使用顺序的建议。

4. **「实验方案设计」和「消融实验设计」有重叠**：前者包含了消融实验的初步设计，后者是更深入的细化。可以考虑让用户先用「实验方案设计」确定大方向，再用「消融实验设计」细化。

5. **论文本身作为输入的场景可以更突出**：目前只有「论文精读笔记」明确标注可以粘贴论文，但实际上「创新点提炼」「语言审查」「一致性审查」都能接受论文内容。可以在 UI 上为这些卡片添加「粘贴论文」的快捷提示。
